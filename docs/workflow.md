### 题目体系设计

**题目类型**：

| 类型 | 数量 | 用途 | 示例 |
|------|------|------|------|
| 场景选择 | 8-10题 | 主要维度测量 | "周末早上你更想..." |

### 隐藏机制设计

**类型1：视觉异常**
- 某道题的文字颜色略有不同
- 某张图片有细微倒置
- 某个选项的位置异常

**类型2：行为追踪**
- 停留时间异常长的题目
- 反复滚动查看的区域
- 点击非交互元素的尝试

**类型3：彩蛋路径**
- 发现异常 → 弹出特殊对话 → 解锁隐藏角色/题目
- 需要输入特定指令才能触发

## Phase 4: 算法实现

### 目标
- 实现评分系统
- 实现角色匹配算法
- 实现数据追踪

### 评分流程

```
用户作答 → 维度得分计算 → 隐藏发现加成 → 最终向量生成 → 角色匹配 → 结果输出
```

### 角色匹配算法

```python
def match_character(user_vector, characters):
    """
    user_vector: 用户在各维度的得分向量
    characters: 角色库，每个角色有理想向量和权重
    """
    scores = []
    for char in characters:
        # 加权余弦相似度
        weighted_user = user_vector * char['weights']
        weighted_char = char['vector'] * char['weights']
        similarity = cosine_similarity(weighted_user, weighted_char)
        scores.append((char, similarity))
    
    # 排序返回最匹配的
    scores.sort(key=lambda x: x[1], reverse=True)
    return scores[0]
```

## Phase 5: 验证迭代

### 目标
- 验证测量准确性
- 优化题目质量
- 完善用户体验

### 验证方法

| 方法 | 样本量 | 周期 | 目标 |
|------|--------|------|------|
| Pilot 测试 | 30-50人 | 1周 | 发现明显问题 |
| 信度检验 | 100+人 | 2周 | Cronbach's α > 0.7 |
| 重测信度 | 50人 | 4周 | r > 0.7 |
| 效标效度 | 100+人 | 与大五量表对照 | 中等相关 |

### 迭代优化

1. 分析哪些题目区分度不够
2. 调整角色向量权重
3. 优化隐藏机制的触发率
4. 完善角色描述和分析文案
